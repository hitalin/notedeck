# 自己プロファイル — 自己拡張するスキル

このスキルは「**AI が NoteDeck の中身を自分で育てる**」の最小デモ。ユーザーの
最近のメモを読んで「**現在の自分像**」を抽出し、**この skill 自身の本文**
の各セクションを **常に最新の状態で書き換える** (=  append ではなく
replaceSection)。

過去ログは memos がそのまま保持している。このスキルが担うのは「**常に最新の
私を要約した薄い 1 ページ**」を維持することだけ。これにより:

- skill body は無限に肥大しない (常に 500〜1000 字程度)
- AI のシステムプロンプトに乗り続けても token を食わない
- ユーザーは skill カラムからいつでも「AI が私をどう見ているか」を読める

ユーザーが skill 本文を見て違和感を感じたら、手で書き換えればよい。次の起動
時に AI はそれを尊重し、書き換え後の状態を起点にして更新する。

## 起動方法 (ユーザー判断)

このスキルは「いつ動かすか」をユーザーが選べる:

- **明示呼出し**: チャットで `@self-profile` のように呼ぶか、active 化して
  「プロファイル更新して」と頼む
- **HEARTBEAT 自動化**: skill カラムからこのスキルの mode を `heartbeat` に
  切り替えると、HEARTBEAT daemon (要 ON) が tick ごとに自動実行する

skill 本体の責務は「現在の自分像を再生成・書き換える」ことだけ。起動タイミングは
ユーザー設定 (skill mode + HEARTBEAT 設定) の領域。

## 動作 (起動されたとき)

1. **前回状態の取得**: `skills.read` を `id: 'self-profile'` で呼び、自分の
   現在の body を取得する (= 前回までの圧縮された自己像)
2. **新規メモの取得**: `memos.search` を `olderThanDays: 7` で呼び、直近 1
   週間のメモを取得する
3. **再生成判定**:
   - 新メモが 0 件で、前回状態に変化を加える根拠が無い → 5 へ (= 何もしない)
   - 新メモがあるが、新たに伝わる情報が「前回状態に既に書かれている」と判断
     できる → 5 へ
   - 新メモに「前回状態を更新すべき情報」が含まれる → 4 へ
4. **書き換え**: 該当するセクションだけを `skills.replaceSection` で更新する
   - `## 最近の関心` — 直近で頻出する話題・テーマ
   - `## 学習中` — 進行中の学び・調査
   - `## 文体・好み` — 言葉遣い、絵文字頻度、tone 等
   - `## 避けたいこと` — ユーザーが「これ嫌」と言ったこと、AI が触れない方が
     よさそうな話題
   - 各セクションは **5 行以内**、bullet で簡潔に
   - **無関係なセクションは触らない**
5. **完了の返事**:
   - HEARTBEAT 経由で動いていて、書き換えが無かった → `HEARTBEAT_OK`
     (suppression で履歴 0 件)
   - HEARTBEAT 経由で書き換えを行った → 「`## 最近の関心` を更新しました」
     のような短い 1 行報告
   - 明示呼出し → 何を更新したか普通に説明する (HEARTBEAT_OK は使わない)

## 暴走防止

- 1 回の起動で `skills.replaceSection` の呼び出しは **最大 2 セクションまで**
  (全セクション一斉更新は禁止 = 既存内容の上書き暴走を防ぐ)
- 既存セクションを **空にしてはいけない** (= 確信が持てなければ前回値を残す)
- 各セクションは 5 行以内、合計 body は 1000 字を目安に上限維持

## 振り返りモード (書き換えなし)

ユーザーが「プロファイル見せて」「最近の自分は?」と聞いた場合は、書き換えは
行わず **現在の body の要約** を返す。書き換えは「最新化して」「memo から
更新して」のような明示的依頼があったときだけ。

## ユーザー編集の尊重

ユーザーが手で skill 本文を編集した場合、AI はそれを「現在の自分像」として
尊重する。次回起動で `skills.read` を呼んだとき得られるのはユーザー編集後の
内容なので、自然にその上に更新が乗る。**勝手にロールバックしない**。
