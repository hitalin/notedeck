# Tauri V2 vs Slint 評価レポート

## 概要

notedeck（マルチサーバー対応 Misskey デッキクライアント）の技術選定として、Tauri V2 と Slint のどちらが適切かを評価する。

## 現在の構成

- **フレームワーク**: Tauri V2
- **フロントエンド**: Vue 3 + TypeScript + Vite
- **バックエンド**: Rust (Axum, notecli)
- **対応プラットフォーム**: Windows, macOS, Linux, Android (開発中)

## 比較

### Tauri V2

**長所:**
- Web技術（HTML/CSS/JS）による高い表現力とUI柔軟性
- npm/Vue/Reactの巨大なエコシステムを活用可能
- デスクトップ + モバイル対応が成熟
- HMR（Hot Module Replacement）による高速な開発サイクル
- MITライセンス（完全自由）
- WebViewベースでリッチコンテンツ（画像、動画、OGP等）の表示が容易

**短所:**
- WebViewに依存（OSごとに挙動差あり）
- ネイティブ描画に比べメモリ消費が大きい
- WebView起動のオーバーヘッド

### Slint

**長所:**
- 独自の宣言型UI言語（.slint）でネイティブ描画
- 組み込みデバイスでも動作（RAM 300KiB以下）
- 複数レンダリングバックエンド（OpenGL ES, Skia, ソフトウェア）
- Rustとの深い統合
- AOTコンパイルによる高い実行効率

**短所:**
- デスクトッププラットフォームサポートが「In Progress」
- 組み込み・IoT向けが主なターゲット
- UIコンポーネントが限定的（Web技術に比べ）
- リッチコンテンツ（動画、OGP、絵文字等）の表示が困難
- GPLv3 / 商用デュアルライセンス

## 評価

| 観点 | Tauri V2 | Slint | 重要度 |
|------|----------|-------|--------|
| UIの柔軟性（デッキUI） | ◎ | △ | 高 |
| エコシステム | ◎ | △ | 高 |
| プラットフォーム対応 | ◎ | ○ | 高 |
| リッチコンテンツ表示 | ◎ | △ | 高 |
| 開発効率 | ◎ | ○ | 中 |
| ライセンス自由度 | ◎ | △ | 中 |
| メモリ効率 | ○ | ◎ | 低 |
| ネイティブ描画 | △ | ◎ | 低 |

## 結論

**Tauri V2 が正しい選択である。**

notedeck は SNS デッキクライアントであり、以下の要件から Tauri V2 が明確に適している：

1. **マルチカラムデッキUI**: CSS Grid/Flexbox による複雑なレイアウトが必要
2. **リッチコンテンツ**: OGPプレビュー、画像/動画表示、カスタム絵文字、リッチテキスト
3. **プラグインシステム**: AiScript 等のスクリプト実行環境
4. **エコシステム活用**: CodeMirror, Shiki 等の成熟したWebライブラリ
5. **モバイル展開**: Tauri V2 のモバイルサポートが実用段階

Slint は組み込み・IoT デバイス向けに優れたツールキットだが、リッチな SNS クライアントのユースケースには適さない。
