# NoteDeck アーキテクチャ採用状況 & 未採用の革新的パターン

## 凡例

- **採用済** = 現在のコードベースで実装済み
- **部分採用** = 基盤はあるが限定的な利用
- **未採用（推奨）** = 導入すれば革新的なUXを提供できるもの

---

## 1. 採用済みアーキテクチャ（7つ）

### 1-1. Query Bridge（Rust↔フロントエンド双方向クエリ）
**場所**: `query_bridge.rs` + `utils/apiBridge.ts`

外部 HTTP リクエスト → Rust → Tauri Event → Vue/Pinia → Tauri Event → Rust → HTTP レスポンス。
フロントエンドのリアクティブ状態（デッキカラム、コマンド一覧等）を外部ツールから直接取得可能。

**UX効果**: curl や AI エージェントから NoteDeck を操作可能。

---

### 1-2. マルチウィンドウ・デッキ（クロスウィンドウ D&D）
**場所**: `useDeckWindow.ts` + `useColumnDrag.ts`

- カラムを別ウィンドウにポップアウト
- ウィンドウ間でカラムをドラッグ移動（`deck:drag-start` / `deck:move-column` イベント）
- マルチモニター対応のレイアウト保存・復元
- ウィンドウ閉鎖時の自動カラム回収

**UX効果**: マルチモニターでの本格的なデッキ運用。TweetDeck を超えるカスタマイズ性。

---

### 1-3. 二重 API（IPC + localhost HTTP）
**場所**: `commands.rs`（90+コマンド）+ `http_server.rs`（39ルート + OpenAPI）

同一ロジックを Tauri IPC（アプリ内）と Axum HTTP（外部）の2つのインターフェースで公開。
SSE イベントストリーム、OpenAPI ドキュメント、Scalar UI 付き。

**UX効果**: 外部スクリプト・AIエージェント・CLIツールとの連携。

---

### 1-4. ストリーミング→マルチ配信ブリッジ
**場所**: `streaming.rs` + `EventBus`

WebSocket 受信 → 1箇所で3つの出力先に同時配信:
1. OS ネイティブ通知（`tauri-plugin-notification`）
2. WebView イベント（`app.emit("stream-event")`）
3. SSE（外部 HTTP クライアント向け）

**UX効果**: アプリを開いていなくてもリアクション・リプライを見逃さない。

---

### 1-5. 3層画像プロキシキャッシュ
**場所**: `image_cache.rs` + `/proxy/image`

メモリ LRU（32MB）→ ディスク（24h TTL）→ ネットワーク（ストリーミング配信）。
ETag / 304 対応、インフライト重複排除、同時フェッチ20件制限。

**UX効果**: 画像の瞬間表示。帯域節約。外部サーバー障害時もキャッシュ済み画像を表示。

---

### 1-6. OGP プラグインシステム（15プラットフォーム対応）
**場所**: `ogp/plugins/` (Twitter, YouTube, Pixiv, Amazon, ニコニコ 等)

URL ごとに専用パーサーが起動し、汎用 OG タグ解析より高精度なプレビューを生成。
3段フォールバック: プラグイン → サーバー API → 直接 HTML パース。

**UX効果**: YouTube はサムネ+再生時間、Amazon は価格表示、Twitter は全文表示等リッチなプレビュー。

---

### 1-7. グローバルショートカット + ボスキー + システムトレイ
**場所**: `lib.rs`（デスクトップ専用 `#[cfg(not(mobile))]`）

- Ctrl+Shift+B: ボスキー（瞬時にウィンドウ非表示）
- Ctrl+Alt+N: クイックノート（ウィンドウ表示 + 投稿フォーム起動）
- トレイアイコン: 左クリックで表示切替、右クリックメニュー
- 閉じるボタン: トレイに隠す（終了しない）

**UX効果**: 常駐型の SNS クライアントとして自然な操作感。

---

## 2. 部分採用のアーキテクチャ（3つ）

### 2-1. カスタム URI スキーム（`notedeck://`）
**現状**: `deck.ts` で URI を生成しタイトルバーに表示するのみ。
**足りないもの**: OS レベルのディープリンク登録。ブラウザや他アプリから `notedeck://misskey.io/notes/xxx` でノートを直接開く機能。

---

### 2-2. Android バックグラウンドポーリング
**現状**: `NotificationWorker.kt` で15分間隔の通知ポーリングを実装済み。
**足りないもの**: WebSocket ベースのフォアグラウンドサービスによるリアルタイム通知（15分遅延の解消）。

---

### 2-3. AiScript プラグインシステム
**現状**: Web Worker で LSP 搭載のコードエディタ、プラグイン実行エンジン、UI レンダラーまで実装済み。
**足りないもの**: プラグインマーケットプレイス（共有・インストールの仕組み）。

---

## 3. 未採用だが革新的な UX を提供できるアーキテクチャ（8つ）

### 3-1. オフラインファースト・アーキテクチャ ★★★
**概要**: SQLite キャッシュを拡張し、オフライン時もタイムライン閲覧・ノート下書き・リアクション予約を可能にする。

**Tauri V2 での実装**:
```
オンライン: WebSocket → ノート受信 → SQLite 保存 → UI 表示
オフライン: SQLite から復元 → UI 表示 → 操作をキューに蓄積
復帰時:    キューを一括送信 → 競合解決 → UI 同期
```

**現状との差分**: 現在は `api_get_cached_timeline` でキャッシュ読み出しが可能だが、**書き込み操作のキューイング**と**オフライン検出→自動切替**がない。

**UX効果**: 電車の中やWi-Fi不安定な場所でもタイムラインを快適に閲覧。下書きを書いておいて接続復帰時に自動投稿。

---

### 3-2. ディープリンク（OS統合）★★★
**概要**: `notedeck://` スキームを OS に登録し、ブラウザやチャットアプリで Misskey の URL をクリックすると NoteDeck で直接開く。

**Tauri V2 での実装**:
```
tauri.conf.json に deep-link プラグイン設定
↓
OS がURL をキャプチャ → Tauri がイベント発火
↓
フロントエンドが該当カラム/ウィンドウを開く
```

**活用例**:
- `notedeck://misskey.io/notes/xxxx` → ノート詳細を開く
- `notedeck://misskey.io/user/alice` → ユーザープロフィールを開く
- ブラウザで misskey.io のリンクをクリック → NoteDeck で開く設定

**UX効果**: Web と デスクトップアプリの境界がなくなる。

---

### 3-3. ウィンドウ透過・ブラー効果 ★★
**概要**: Tauri V2 の Window Effects API でウィンドウ背景にすりガラス効果を適用。

**Tauri V2 での実装**:
```rust
// Tauri V2 Window Effects
WindowEffectsConfig {
    effects: vec![WindowEffect::Mica],  // Windows 11
    // or WindowEffect::Vibrancy         // macOS
    state: Some(WindowEffectState::Active),
    ..Default::default()
}
```

**UX効果**: macOS の Vibrancy / Windows 11 の Mica のようなネイティブ感のある美しいUI。デスクトップ壁紙が透けて見えるデッキ。

---

### 3-4. クリップボード統合 ★★
**概要**: `tauri-plugin-clipboard-manager` で画像やリッチテキストのクリップボード操作を強化。

**活用例**:
- スクリーンショットを撮影 → Ctrl+V で即座に投稿フォームに添付
- ノートのテキストをリッチコピー（MFM 装飾付き）
- 他アプリから画像コピー → ドライブに直接アップロード

**UX効果**: 「スクショ → 貼り付け → 投稿」が3秒で完了する。

---

### 3-5. Sidecar + ローカル AI 統合 ★★★
**概要**: Tauri の sidecar 機能で Ollama やローカル LLM を起動し、NoteDeck 内でAI機能を提供。

**活用例**:
- タイムラインの要約（「今日の misskey.io で何があった？」）
- 投稿の翻訳・校正（ノート作成時にAIが提案）
- コンテンツフィルタリング（不快なノートを自動非表示）
- リアクション提案（ノート内容に基づく絵文字推薦）

**Tauri V2 での実装**:
```rust
// sidecar でローカル LLM を起動
tauri::api::process::Command::new_sidecar("ollama")
    .args(["serve"])
    .spawn()

// HTTP経由でLLMに問い合わせ
reqwest::post("http://localhost:11434/api/generate")
    .json(&json!({ "model": "llama3", "prompt": summary_prompt }))
```

**UX効果**: クラウドに送信せずプライバシーを保ちながら AI 機能を利用。

---

### 3-6. ファイル関連付け + ドラッグ&ドロップ統合 ★
**概要**: `.msky` や `.notedeck` などの独自ファイル形式を OS に登録。デッキプロファイルの共有をファイルベースで実現。

**活用例**:
- デッキレイアウトを `.notedeck` ファイルとしてエクスポート → 友人に共有
- ファイルマネージャから画像をデッキにドロップ → 自動投稿
- AiScript プラグインファイル (.as) をダブルクリック → 自動インストール

**UX効果**: OS レベルのファイル操作で設定共有。

---

### 3-7. マルチプロセス・サンドボックス化プラグイン ★★
**概要**: AiScript プラグインを独立した WebView ウィンドウ(iframe)ではなく、Tauri の別プロセスとして隔離実行。

**Tauri V2 での実装**:
```
メインプロセス → サンドボックス化 WebView (プラグインごと)
                 ├─ 制限された IPC のみ許可
                 ├─ ネットワーク・ファイルアクセス不可
                 └─ API はホワイトリスト方式で公開
```

**UX効果**: 悪意のあるプラグインからのデータ漏洩を防止。安心してサードパーティプラグインを利用。

---

### 3-8. P2P デッキ同期（WebRTC / Tauri IPC）★
**概要**: 同一 LAN 上の複数デバイス間でデッキ状態をリアルタイム同期。

**活用例**:
- デスクトップ PC のデッキレイアウトがラップトップに自動反映
- スマホで書いた下書きがデスクトップに即座に現れる
- マルチデバイスでの「続きから」体験

**UX効果**: クラウド不要のローカルデバイス間同期。プライバシーを保ちながらシームレスなマルチデバイス体験。

---

## まとめ

```
採用状況マトリクス:

                        実装済み        部分採用        未採用（革新的）
                        ────────        ────────        ────────────────
Rust↔Frontend通信    │ Query Bridge  │               │
マルチウィンドウ      │ クロスW D&D   │               │
外部API公開          │ 二重API        │               │
リアルタイム通信      │ マルチ配信     │ Android BG    │
キャッシュ           │ 3層画像/OGP   │               │ オフラインファースト
OS統合               │ トレイ/ショートカット │ URI スキーム │ ディープリンク, 透過, クリップボード
プラグイン           │               │ AiScript      │ サンドボックス, マーケット
AI連携               │               │               │ Sidecar LLM
デバイス間同期       │               │               │ P2P同期
```

**最も投資対効果が高い未採用パターン（推奨順）**:
1. **オフラインファースト** — 既存の SQLite キャッシュ基盤を拡張するだけで実現可能
2. **ディープリンク** — Tauri プラグイン追加 + フロントエンドルーティングで実現可能
3. **Sidecar ローカル AI** — Ollama 連携で差別化された UX を提供
